# Purpose

Hono + Remult backend providing the REST endpoints, live query database sync, and agent execution control.
Multi-user authentication via better-auth with email/password.

# Ownership

- Gateway Backend Dev Team / Gateway Backend Agent

# Local Contracts

- Port: Runs on port `3001`.
- Database: Interacts with `db.sqlite` at the repo root using `remult/remult-bun-sqlite`.
- API Security: Key management is encrypted via AES-256-GCM server-side (`encryption.ts`).
- Workspace: Agent tools (read, write, edit, search, bash) are restricted/scoped to `~/.stratum/workspace`.
- Headroom: Auto-managed proxy resides in `~/.stratum/headroom-venv`.
- CORS: Restricts origin to `http://localhost:5173` (dev).
- Auth: `better-auth` with email/password, mounted at `/api/auth/*`.
- Auth Adapter: `@nerdfolio/remult-better-auth` — bridges better-auth to Remult repositories.
- Auth Tables: `user`, `session`, `account`, `verification` — generated by `src/auth-schema.ts`.
- Auth Entities: `User`, `Session`, `Account`, `Verification` — Remult entity classes in `src/auth-schema.ts`.
- Auth Env Vars: `BETTER_AUTH_SECRET` (required), `BETTER_AUTH_URL` (default: `http://localhost:3001`).
- Remult `getUser`: Calls `auth.api.getSession({ headers })` to resolve authenticated user.

# Work Guidance

- Register Remult entities and controllers in `src/api.ts`.
- Expose controller routes or CRUD endpoints via `@BackendMethod` in `src/agent-service.ts`.
- Manage CORS and health check endpoints in `src/index.ts`.
- Securely encrypt and decrypt keys using the functions in `src/encryption.ts`.
- **Auth config** lives in `src/auth.ts` — uses `remultAdapter` to share the Remult data provider.
- **Auth schema regeneration**: `bun apps/gateway/src/_gen-schema.ts` (or delete and re-create).
- **New BackendMethods** must be mirrored in `packages/shared/src/controllers/agent-service.ts` with `clientOnly` stubs.
- Entity-level auth: `allowApiCrud: Allow.authenticated` for user-facing entities.
- Session visibility: `ChatSessionSettings.ownerId` + `visibility` for private/shared sessions.

# Verification

- Run type checking: `bunx tsc --noEmit` inside the `apps/gateway` directory.
